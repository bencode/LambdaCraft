# Series 01 · 概率检索与 BM25

BM25 在 2009 年的 Robertson-Zaragoza 综述发布后已经统治 IR 工业界 15 年，在 RAG 时代依然是混合检索的核心组件。但很多人把它当黑盒——只知道 `b=0.75, k1=1.2`，不知道为什么。

## 学完之后你能

- 解释 BM25 公式三层结构（idf / 词频饱和 / 长度归一化）各自从哪个假设来
- 推导从 Probability Ranking Principle 到 BIM 到 2-Poisson 近似到 BM25 的完整链条
- 说清 BM25 vs tf-idf 在哪类 query 上为什么效果不同
- 调 `k1` 和 `b` 两个参数能解释做什么
- 知道 BM25F 等扩展（多字段加权）的设计动机
